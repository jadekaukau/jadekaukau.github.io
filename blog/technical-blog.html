Whether you believe in some divine ruler, G O D or some nerdy silicon valley stork plopping websites on your door step (or google search), there ae two core concepts you need to understand: classes and IDs. These building blocks of web design enable us to give different parts of a site distinct styles based on their unique roles.

WTF is a class?

Think of classes as a way to group certain elements together by shared characteristics or attributes. For example: if your site sells both red shirts and blue shirts you could create a "shirt" class for both types of products so that they look visually cohesive (even though they vary in color).

WTH is an ID?

IDs work differently--instead of grouping elements together they provide unique identifiers for specific pieces on display. So if one product on your shirt selling site is particularly eye catching or has its own distinctive features (like sequins) you might assign it its own ID so that it stands out from everything else.

An easy way to remember is that an ID will always start with a ‘#’ on your CSS and a class will start with ‘.’ which you can see in the above images.

Of course classes and IDs have many practical uses beyond just styling individual items--they help streamline design choices and ensure consistency across an entire website.

To maintain consistency in design across multiple elements in your web page, using classes is vital. If several paragraphs require similar fonts or colors, grouping them into one class name is an efficient way of achieving this goal.

However, if there's only one specific element that needs unique styling compared to others on the page (for example a specific button design), using IDs comes in handy.
